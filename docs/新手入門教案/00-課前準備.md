## 核心觀念

### PHP 版本管理

macOS 可能預裝舊版 PHP，建議使用 Homebrew 管理 PHP 版本：

```bash
# 安裝 PHP 8.4
brew install php@8.4

# 確認版本
php -v
```

若系統有多版本 PHP，可使用 `brew link` 切換：

```bash
brew unlink php
brew link php@8.4 --force
```

### Composer 全域安裝

```bash
# 下載安裝腳本
php -r "copy('https://getcomposer.org/installer', 'composer-setup.php');"

# 執行安裝
php composer-setup.php

# 移至全域可執行路徑
sudo mv composer.phar /usr/local/bin/composer

# 確認安裝
composer -V
```

### Docker Desktop

Docker Desktop 提供圖形化介面管理容器：

1. 下載並安裝 Docker Desktop
2. 啟動應用程式
3. 確認 Docker Engine 已啟動（系統列有鯨魚圖示）

### VS Code 推薦擴充

| 擴充套件 | 用途 |
|---------|------|
| PHP Intelephense | PHP 智慧提示與自動完成 |
| Laravel Blade Snippets | Blade 模板語法支援 |
| Docker | Docker 檔案支援 |
| GitLens | Git 增強功能 |
| EditorConfig | 統一編輯器設定 |

## 驗證指令

```bash
# 1. 確認 PHP 版本
php -v
# 預期輸出：PHP 8.4.x

# 2. 確認 PHP 擴充
php -m | grep -E "(pdo|mysql|redis|mbstring|openssl|curl)"
# 預期輸出：pdo, pdo_mysql, redis（或 phpredis）, mbstring, openssl, curl

# 3. 確認 Composer
composer -V
# 預期輸出：Composer version 2.x.x

# 4. 確認 Docker Compose
docker compose version
# 預期輸出：Docker Compose version v2.x.x

# 5. Clone 專案並安裝依賴
git clone <本 Repo URL> && cd learn-php-laravel
composer install
# 預期輸出：Installing dependencies from lock file

# 6. 啟動容器
docker compose up -d
# 預期輸出：Container ... Started

# 7. 確認容器狀態
docker compose ps
# 預期輸出：三個容器都是 running 狀態
```

## 踩坑提示

| 現象 | 原因 | 解法 |
|------|------|------|
| `docker: command not found` | Docker Desktop 未啟動或未安裝 | macOS / Windows 需先啟動 Docker Desktop 應用程式 |
| Composer 下載依賴卡住或逾時 | 網路問題或 Packagist 連線慢 | 使用中國鏡像：`composer config -g repo.packagist composer https://mirrors.aliyun.com/composer/` |
| `php -v` 顯示非 8.4 | 系統有多版本 PHP | macOS 用 `brew link php@8.4 --force`；或設定 PATH |
| Extension 缺失（如 pdo_mysql） | PHP 安裝時未啟用擴充 | `brew install php@8.4` 預設已含常用擴充；或手動安裝 |
| Permission denied | 權限問題 | 使用 `sudo` 或調整資料夾權限 |
| WSL2 中 Docker 無法連線 | Docker Desktop 未啟用 WSL2 整合 | 在 Docker Desktop Settings → Resources → WSL Integration 啟用 |
