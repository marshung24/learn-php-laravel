## 對應檔案

| 檔案 | 角色 |
|------|------|
| `.env` | 環境變數檔（本機開發用） |
| `.env.example` | 環境變數範本 |

## 核心觀念

### `.env` 環境變數

敏感資訊（密碼、API Key）不進版控，用 `.env` 管理：

```env
APP_NAME="書籍管理系統"
APP_ENV=local
APP_DEBUG=true

DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=app
DB_USERNAME=app
DB_PASSWORD=app

REDIS_HOST=127.0.0.1
REDIS_PORT=6379

CACHE_DRIVER=redis
SESSION_DRIVER=redis
```

**重要**：`.env` 不進版控（已在 `.gitignore`），每個環境有自己的 `.env`。

### `env()` 讀取環境變數

```php
// 讀取 .env 中的 DB_HOST，預設值為 localhost
$host = env('DB_HOST', 'localhost');
```

### `config()` 讀取設定

```php
// 讀取 config/database.php 的 default 值
$driver = config('database.default');

// 讀取巢狀設定
$host = config('database.connections.mysql.host');
```

### config 檔案結構

```php
// config/database.php
return [
    'default' => env('DB_CONNECTION', 'mysql'),

    'connections' => [
        'mysql' => [
            'driver' => 'mysql',
            'host' => env('DB_HOST', '127.0.0.1'),
            'port' => env('DB_PORT', '3306'),
            'database' => env('DB_DATABASE', 'forge'),
            'username' => env('DB_USERNAME', 'forge'),
            'password' => env('DB_PASSWORD', ''),
            // ...
        ],
    ],
];
```

### 設定優先順序

```
環境變數 > .env 檔案 > config 預設值
```

### 設定快取

```bash
# 快取設定（正式環境用，加速載入）
php artisan config:cache

# 清除設定快取（開發時修改 .env 後必須執行）
php artisan config:clear

# 清除所有快取
php artisan cache:clear
```

**注意**：執行 `config:cache` 後，`env()` 只能在 `config/` 檔案中使用，其他地方會回傳 `null`。

### Docker Compose 一鍵啟動

```yaml
# docker-compose.yml
version: '3.8'

services:
  app:
    build:
      context: .
      dockerfile: docker/php/Dockerfile
    ports:
      - "8000:8000"
    volumes:
      - .:/var/www/html
    depends_on:
      - db
      - redis

  db:
    image: mysql:8.0
    environment:
      MYSQL_DATABASE: app
      MYSQL_USER: app
      MYSQL_PASSWORD: app
      MYSQL_ROOT_PASSWORD: root
    ports:
      - "3306:3306"
    volumes:
      - mysql_data:/var/lib/mysql

  redis:
    image: redis:7-alpine
    ports:
      - "6379:6379"

volumes:
  mysql_data:
```

## 動手做

### 必做

**1. 啟動 Docker Compose**

```bash
# 啟動所有容器（背景執行）
docker compose up -d

# 確認容器狀態
docker compose ps
# 預期：三個容器（app, db, redis）都是 running

# 查看容器 log
docker compose logs -f db
```

**2. 理解 .env 與 config 的關係**

比對 `.env` 中的設定：

```env
DB_HOST=127.0.0.1
```

與 `config/database.php` 中的對應：

```php
'host' => env('DB_HOST', '127.0.0.1'),
```

說明 `env()` 如何從 `.env` 讀取值，以及預設值的作用。

**3. 測試資料庫連線**

```bash
# 使用 artisan 測試連線
php artisan tinker

>>> DB::connection()->getPdo();
# 成功會回傳 PDO 物件

>>> exit
```

## 踩坑提示

| 現象 | 原因 | 解法 |
|------|------|------|
| 改了 `.env` 但沒生效 | 設定被快取 | `php artisan config:clear` 清除快取 |
| `docker compose up` 後 App 連不上 DB | MySQL 容器尚未 ready | 等幾秒讓 MySQL 初始化完成；或查看 `docker compose logs db` 確認狀態 |
| `.env` 變數格式錯誤 | 值包含空格或特殊字元未加引號 | 用雙引號包住值，如 `APP_NAME="My App"` |
| `env()` 在程式碼中回傳 null | 執行了 `config:cache` | 只在 `config/` 檔案中使用 `env()`；其他地方改用 `config()` |
| 容器無法存取 host 的 localhost | Docker 網路隔離 | 容器間用服務名稱（如 `db`）而非 `localhost` |
